# 1주차 화면 구성표 v0.4

## 1. 화면 구성표 작성 목적

본 문서는 Mirae Mobility 팀의 「독수리 택시」 프로젝트에서 React 프론트엔드가 구현해야 할 화면 구조를 정리한 내부 개발 기준 문서이다.

화면 구성표는 단순한 디자인 문서가 아니라, 각 화면의 목적, 접근 권한, 입력 요소, 표시 데이터, 버튼, 연결 API, 예외 처리, 완료 기준을 정의한다.

본 문서는 기능명세서, API 명세서, 알고리즘 설계서, 운영 계획(안), 개발 계획(안)의 최신 기준을 따른다.

### 입력값 기준

<!-- 표 8 -->
| 화면 표시 | API 필드 | 기준 |
| --- | --- | --- |
| 이름 또는 닉네임 | name | 필수 |
| 이메일 | email | 필수 |

성별은 성별 매칭 옵션 판단에 사용된다.

개인정보 안내는 /legal에서 확인할 수 있도록 연결한다.

## S-03. 로그인

<!-- 표 9 -->
| 항목 | 내용 |
| --- | --- |
| 화면명 | 로그인 |
| 화면 ID | S-03 |
| URL Path | /login |
| 화면 목적 | 사용자가 이메일과 비밀번호로 로그인한다. |
| 접근 권한 | 비회원 |
| 입력 요소 | 이메일, 비밀번호 |
| 표시 데이터 | 로그인 입력 폼, 회원가입 이동 링크 |
| 버튼 | 로그인하기, 회원가입 화면으로 이동 |
| 연결 API | POST /api/auth/login |
| 성공 시 동작 | access_token 저장, GET /api/auth/me로 사용자 상태 확인, /main 화면으로 이동 |
| 실패 시 동작 | “이메일 또는 비밀번호가 올바르지 않습니다.” |
| 화면 완료 기준 | 로그인 성공 시 access_token이 저장되고 /api/auth/me로 사용자 상태를 확인한 뒤 /main 화면으로 이동되어야 함 |

주의사항:

실패 문구는 이메일 오류와 비밀번호 오류를 구분하지 않는다.

access_token은 localStorage에 저장한다.

localStorage는 로그인 토큰 저장 외 용도로 사용하지 않는다.

### 자동 계산 기준

### 자동 계산 기준

출발지, 도착지, 희망 출발 시간이 모두 선택되면 GET /api/fares/estimate를 호출해 예상 택시비를 자동 산정한다.

출발지, 도착지, 희망 출발 시간 중 하나라도 변경되면 예상 택시비와 1인 예상 요금을 다시 자동 계산한다.

사용자는 예상 택시비를 직접 입력하지 않는다.

사용자는 1인 예상 요금을 직접 입력하지 않는다.

파티 생성 직후 current_members=1이다.

파티 생성 직후 per_person_fare=estimated_fare이다.

예상 택시비 산정에 실패하면 파티 생성은 차단된다.

“예상 요금 확인” 버튼은 필수 버튼으로 두지 않는다. 필요한 경우 보조 갱신 버튼으로만 사용할 수 있다.

# 6. 화면 구성표 최종 점검표

□ 요금/결제 화면이 삭제되었는가

□ 요금 정보가 파티 상세 화면에 포함되었는가

□ 랜딩 페이지가 추가되었는가

□ 메인 페이지가 추가되었는가

□ 내 파티 목록 화면이 추가되었는가

□ 안전 안내 화면이 추가되었는가

□ 계정·설정 화면이 추가되었는가

□ 이용약관과 개인정보 처리방침 화면이 분리되어 추가되었는가

□ 관리자 페이지가 추가되었는가

□ 파티 생성 화면에서 GET /api/places가 제거되었는가

□ 파티 생성 화면에 Kakao Maps Places keywordSearch가 반영되었는가

□ 예상 택시비와 1인 예상 요금이 자동 계산 기준으로 정리되었는가

□ 파티 상세 URL이 /parties/:partyId로 수정되었는가

□ 채팅 URL이 /parties/:partyId/chat으로 정리되었는가

□ 채팅 접근 권한이 파티 참여자로 제한되었는가

□ 신고 버튼은 유지하되 신고 저장 API는 제외되었는가

□ 내 파티 목록이 GET /api/my/parties와 연결되었는가

□ 관리자 페이지가 확장된 관리자 API와 연결되었는가

□ 성별 입력이 남성/여성 필수 선택으로 정리되었는가

□ 이동 시간대 중복 참여 차단 기준이 반영되었는가
